📖 How to Use This Map

This interactive map shows all 85 data tables in the Laynhapuy ERP system and how they connect to each other. Use it to understand which tables you need to join when building reports.

🖱️ Exploring the Map

Click any table (circle) to highlight all its connections. Click it again or click empty space to deselect.

Hover over any table to see its details — name, module, field count, description, and all relationships.

Scroll to zoom in and out. Drag the background to pan around.

Drag any table circle to rearrange the layout.

🔍 Searching

Type in the search box (top right) to find a table by name. Click a result to zoom to that table and highlight its connections.

📂 Module Filters

Use the buttons at the top to show only tables from a specific module area. "All" shows everything. The filter also shows directly connected tables from other modules so you can see cross-module relationships.

🔗 Join Path Finder

This is the key tool for report building. Click 🔗 Join Finder to open it, then select two tables to see exactly how they connect — which intermediate tables you need to join through.

Example: "I want to show the creditor name on a purchase order."
→ Select Purchase Orders → Creditor Master
→ Result: PO → CRS (via Creditor Code) — a direct join!
Example: "I want to see which GL accounts an employee's pay was charged to."
→ Select Pay Work History → Chart of Accounts
→ Result: PAYWORK_HIST → COA (via Cost Centre) — direct join!

🎨 What the Colours Mean

Each colour represents a module: ■ General Ledger, ■ Payroll, ■ Creditors, ■ Debtors, ■ Assets, ■ Plant, ■ Maintenance, ■ Stock, ■ Work Orders, ■ Grants.

Bigger circles = more data fields. The largest ones (Payroll Master, Asset Register) have 100+ fields each.

💡 Tips

The Chart of Accounts (COA) is the central hub — nearly every module connects to it for GL posting. If you're building a financial report, COA is almost always in the join path.
